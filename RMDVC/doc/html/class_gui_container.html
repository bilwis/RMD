Detailed Description

A class that can hold several GuiElement children.

It provides Element management and rendering functions. It is intended to be used as a sort of 'Panel' which can be moved, and that holds several GuiElement children which are drawn on it.

Member Function Documentation

void GuiContainer::render ( TCODConsole *  con)
virtual

Renders all 'child' GuiElements on the given console.

This function calls the render() function of all GuiElements contained in the elements vector, after calling renderSelf() . These functions draw on the container_console, which is then blitted onto the given console.

Implements RenderObject.

Reimplemented in GuiBodyViewer.

void GuiContainer::renderSelf ( TCODConsole *  con)

Renders only the GuiContainers own graphic elements onto the given console.

This function renders only the GuiContainers own graphic elements (such as the border). It does not call the render() function of it's children to allow for custom rendering (such as with ActiveElements). Note that this function should be called before calling the child render function(s), otherwise it will overwrite the output on the console.

Member Data Documentation

TCODConsole* GuiContainer::container_console
protected

The GuiContainer's TCODConsole.

Every GuiContainer declares it's own TCODConsole, on which it renders its GuiElement children. The Console is then passed to the instance of the Gui class handling the overall Gui rendering, which blits it onto the root console.

bool GuiContainer::dynamic
protected

A dynamic GuiContainer is marked to be updated every turn by the Gui handling class. This should only be used for integral Gui parts such as health bars etc.

std::vector<GuiElement*>* GuiContainer::elements
protected

The vector holding pointers to the 'child' elements. Note: Nesting GuiContainers (i.e. having a GuiContainer element be a child of a GuiContainer) is not supported
